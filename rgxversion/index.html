flask源码阅读笔记~正则自动化

2016-07-19 23:55:16

flask中正则表达式的应用

说到正则表达式, 大家第一印象, 这货是用来进行字符串工作的: 匹配、查找、替换…
然而正则的灵活以及对字符串的处理能力, 可以帮助我们进行一些繁琐的工作, 比如:
更换版本号.

使用正则表达式自动更换版本号

还记得我开发mana的时候, 每发布一个新的版本, 都要先去setup.py中修改版本号,
然后再去mana version命令中再次修改版本号输出, 很是麻烦. 而flask源码中,
只在__init__.py作为版本号的源,
其他地方(比如setup.py)则使用正则从源自动更新版本号, 所以每次更新版本,
只要更新一处就行了!

version正则表达式

  • 源: flask,__init__.py: version = ‘0.11.2-dev’
  • 正则: flask, setup.py: _version_re = re.compile(r’version\s+=\s+(.*)’)

解释一下这个正则表达式'__version__\s+=\s+(.*)', 其实很简单:
__version__匹配__version__, \s+匹配=前的空格, =匹配=,
\s+匹配等号右侧的空格, (.*)匹配版本号也就是'0.11.2-dev'.

然后从__init__.py中读取源, 就可以匹配()中的版本号了

with open('flask/__init__.py', 'rb') as f:
    version = str(ast.literal_eval(_version_re.search(
        f.read().decode('utf-8')).group(1)))

这里使用_version_re.search搜索匹配, 并返回group(1).
search时需要把utf-8编码的文件转化为unicode.

ast.literal_eval的功能和eval类似,
不过会先判断被计算的内容是不是合法的python类型, 所以str(ast.literal_eval(version))
的作用就是先检查version的类型是否合法, 然后转为python str字符串的形式.
可以看出flask不打算迁移python3:)
